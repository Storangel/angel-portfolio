# Angel Portfolio

## Vision

Angel Portfolio is a professional digital platform that represents the complete professional identity of its owner — not merely a collection of projects, but a deliberate expression of expertise as an **IT Manager**, **Software Developer**, and **Digital Transformation Specialist**.

The website serves as a strategic communication tool. It must convey **trust**, **technical excellence**, and **business understanding** to every visitor within seconds of arrival. Visitors should immediately perceive a professional who bridges the gap between technology and business: someone who can architect solutions, lead teams, and drive meaningful organizational change.

This platform is designed to function as a living professional presence — authoritative, credible, and human — that supports career growth, client acquisition, and long-term personal branding in the technology and industrial sectors.

---

## Objectives

The primary goals of Angel Portfolio are:

- **Professional personal brand** — Establish a consistent, memorable identity across digital channels that reflects seniority, reliability, and expertise.
- **Technology portfolio** — Present a curated selection of real projects that demonstrate technical depth, architectural thinking, and delivery capability.
- **Interactive resume** — Offer an engaging, navigable career narrative that goes beyond a static PDF, allowing visitors to explore experience, skills, and achievements contextually.
- **Recruiter presentation** — Provide recruiters with a fast, scannable overview of qualifications, career trajectory, and technical fit for senior roles.
- **Client presentation** — Communicate business value, transformation experience, and delivery credibility to potential clients and decision-makers.
- **Showcase real projects** — Highlight completed work with clear problem statements, solutions, technologies used, and measurable outcomes where applicable.
- **Technical blog (future)** — Reserve architecture and content structure for long-form technical writing, thought leadership, and industry insights.
- **Downloadable CV** — Offer a professionally formatted, up-to-date curriculum vitae for offline sharing and application workflows.
- **Contact channel** — Provide a clear, low-friction path for professional inquiries, collaboration opportunities, and business contact.

## Design Principles

Design inspiration is drawn from industry-leading digital products known for clarity and craft:

- **Apple** — Precision, whitespace, and product-focused presentation
- **Vercel** — Developer-centric elegance and performance-conscious design
- **Linear** — Focused interfaces, purposeful motion, and information hierarchy
- **Stripe** — Trust-building aesthetics and exceptional documentation-style clarity
- **Raycast** — Modern, technical, and refined visual language

### Core Characteristics

- **Lots of white space** — Breathing room between sections to reduce cognitive load and elevate perceived quality
- **Excellent typography** — A deliberate type system with clear hierarchy, readable body text, and purposeful font pairing
- **Simple layouts** — Grid-based, predictable structures that prioritize content over chrome
- **High readability** — Comfortable line lengths, sufficient contrast, and scannable content blocks
- **Smooth animations** — Subtle, purposeful motion that guides attention without distraction
- **No visual overload** — Limited color palette, restrained iconography, and consistent visual rhythm throughout

## SEO Strategy

Angel Portfolio is a personal branding asset. Search visibility directly impacts professional opportunities.

### Focus Areas

| Area | Approach |
|---|---|
| **Personal branding** | Consistent naming, professional metadata, and authoritative content across all pages |
| **Technical SEO** | Semantic HTML, clean URL structure, XML sitemap, and robots.txt configuration |
| **Structured metadata** | JSON-LD schema for Person, ProfessionalService, and Article (future blog) |
| **Open Graph** | Rich social sharing previews with custom OG images per page |
| **Performance** | Core Web Vitals as a ranking signal — fast sites rank better and convert better |
| **Semantic HTML** | Proper heading hierarchy, landmark regions, and meaningful element usage |

### Content SEO Guidelines

- Each page has a unique, descriptive `<title>` and meta description
- Headings follow a logical H1 → H2 → H3 structure without skips
- Images include descriptive alt text
- Internal linking connects related content naturally
- Future blog content targets relevant long-tail keywords in IT management, software development, and digital transformation

## Performance Goals

Performance is a feature, not an afterthought. The site must feel instantaneous.

| Metric | Target |
|---|---|
| **Lighthouse Performance** | ≥ 95 |
| **Lighthouse Accessibility** | ≥ 95 |
| **Lighthouse Best Practices** | ≥ 95 |
| **Lighthouse SEO** | ≥ 95 |
| **LCP (Largest Contentful Paint)** | < 2.5s |
| **FID / INP (Interaction to Next Paint)** | < 200ms |
| **CLS (Cumulative Layout Shift)** | < 0.1 |

### Optimization Strategies

- **Lazy loading** — Defer off-screen images and non-critical resources
- **Optimized images** — Next.js Image component with WebP/AVIF formats and responsive sizing
- **Minimal JavaScript** — Server Components by default; client-side JS only where interactivity is required
- **Font optimization** — Self-hosted or `next/font` with `display: swap` and subset loading
- **Code splitting** — Route-based and component-based splitting to minimize initial bundle size

## Development Rules

These rules govern all development work on Angel Portfolio. They are non-negotiable.

### Architecture

- **Reusable components** — Build once, compose everywhere; no one-off UI implementations
- **Clean architecture** — Separate concerns: data, presentation, and business logic live in distinct layers
- **Scalable folder structure** — Organize by feature and responsibility, not by file type alone
- **No duplicated code** — Extract shared logic into utilities, hooks, or components at the first sign of repetition

### Code Quality

- **Strong TypeScript typing** — No `any`; explicit types for props, API responses, and data models
- **Maintainable code** — Readable naming, small functions, and self-documenting structure over excessive comments
- **Professional commit messages** — Conventional commits format; each commit represents a single logical change

### Non-Functional Requirements

- **Accessibility first** — Accessibility is designed in, not patched in after launch
- **SEO first** — Every page ships with complete metadata and semantic markup
- **Performance first** — Measure before and after every significant change; regressions are blockers

### Workflow

- All changes go through version control on GitHub
- Deployments are automated via Vercel on merge to main
- No direct edits to production without a corresponding commit
- Documentation lives alongside code — this document is the source of truth for product direction
